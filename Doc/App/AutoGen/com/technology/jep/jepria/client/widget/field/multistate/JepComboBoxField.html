Поле для ввода из выпадающего списка.
Основные особенности работы с полем.
Поле в качестве своего значения принимает JepBaseTextField.setValue(Object value) и возвращает JepField.getValue() экземпляр класса JepOption .
Таким образом, для работы с полем необходимо в наследниках JepRecordDefinition установить тип поля JepTypeEnum.OPTION и на серверной стороне в бине делать преобразование из/в JepOption .
Подобный подход в работе со значением поля связан с оптимизацией производительности работы поля - предотвращении lookup'ов (переборе списка значений для получения отображаемого значения) на клиенте и переносе операций на сервер.

Концепция поддержки обработки событий отражена в описании пакета com.technology.jep.jepria.client.widget.

FIRST_TIME_USE_EVENT

Перегружается в наследниках для добавления соответсвующих наследникам полей Gxt.
Особенности:
По умолчанию, в Gxt для комбобокса определено событие срабатывания клавиши, вызывающее метод ComboBox.doQuery(String, boolean).

setViewValue

protected void setViewValue(java.lang.Object value)
Установка значения для карты Просмотра.
При перегрузке данного метода в наследниках необходимо обеспечить, чтобы данный метод был быстрым/НЕ ресурсо-затратным.
Основная идея Jep-полей: они должны быть легкими. Поэтому, карта Просмотра должна быть именно текстовым (или простым Html) представлением значения карты Редактирования.
В тех случаях, когда чисто текстовое представление нецелесообразно (списки, деревья и т.п.) - в поле используется ТОЛЬКО одна карта - карта Редактирования (т.е. карта Просмотра - вообще НЕ используется).
